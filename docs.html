What is BlackRoad OS

Sovereign infrastructure for autonomous AI. A complete operating system built on Raspberry Pis, WireGuard tunnels, and local-first inference -- no cloud dependencies, no vendor lock-in.

BlackRoad OS is a self-hosted platform that combines a fleet of Raspberry Pi nodes with a WireGuard mesh network, local AI inference via Ollama, sovereign Git hosting through Gitea, and a suite of developer tools -- all orchestrated from a single Mac workstation. The system runs 35 AI agents across 5 Pi nodes and 2 cloud droplets, serving 151 domains through a Caddy TLS edge, with 52 TOPS of neural processing power from dual Hailo-8 accelerators.

Unlike traditional cloud platforms, BlackRoad OS follows a zero-dependency philosophy: every service that can be self-hosted is self-hosted. Git repositories live on Gitea instead of GitHub. Object storage runs on MinIO instead of S3. DNS resolves through PowerDNS instead of Cloudflare. The only external dependencies are Stripe for payment processing and GoDaddy for domain registration. Everything else runs on hardware you own.

Architecture

The system follows a hub-and-spoke topology. Your Mac workstation acts as the control plane. All Pi nodes connect through a WireGuard mesh. Internet traffic routes through a Caddy TLS edge on DigitalOcean, then tunnels back to the Pi fleet via WireGuard.

Key Concepts

Agents

35 autonomous AI agents that coordinate through NATS pub/sub messaging and the RoundTrip chat system. Each agent has identity, capabilities, and memory.

Roadies

Every device in the fleet is a Roadie -- a node that contributes compute, storage, or networking to the mesh. Pis, droplets, and your Mac are all Roadies.

Fleet

The collection of all nodes connected via WireGuard. 5 Raspberry Pis, 2 DigitalOcean droplets, and 1 Mac workstation form the current fleet.

Memory

A persistent knowledge system spanning journal entries, codex solutions, TIL broadcasts, and infinite todos. SQLite-backed with FTS5 full-text search.

Quick Install

bash 
$ curl -sSL https://blackroad.io/install | bash

The installer detects your platform, sets up the CLI, configures WireGuard if available, and connects your machine to the fleet as a new Roadie.

Prerequisites

macOS or Linux with bash 4+. WireGuard for mesh connectivity. SSH access to at least one Pi node for full fleet operations.

Installation

Set up BlackRoad OS on your machine and connect to the fleet.

System Requirements

  • Operating System -- macOS 12+ or Linux (Debian/Ubuntu preferred for Pis)
  • Shell -- bash 4+ or zsh
  • Git -- 2.30+
  • SSH -- OpenSSH client with key-based auth configured
  • WireGuard -- required for mesh network connectivity
  • Node.js -- 18+ (for Wrangler and worker tooling)

WireGuard Configuration

The WireGuard mesh connects all nodes with encrypted point-to-point tunnels. Each node gets a static IP on the 10.0.0.0/24 subnet. The mesh currently has 12 active connections across all nodes.

Network Topology

Internet traffic flows: Client → Gematria (Caddy TLS) → WireGuard tunnel → Pi fleet node. All inter-node communication stays within the WireGuard mesh.

RoadChain

Truth ledger. Blockchain-backed integrity verification for the fleet.

Overview

RoadChain is a lightweight blockchain used for logging verifiable truth statements across the fleet. The Solidity smart contract at roadchain/ledger.sol defines the on-chain truth ledger. Agents can submit and verify integrity claims that are immutably recorded.

Identity Hooks

The roadchain-identity.sh script and .roadchain-hooks/ directory provide Git commit hooks that sign commits with RoadChain identity verification. This creates an auditable chain of every code change across the fleet.

Pi Fleet

5 Raspberry Pi 4 nodes running Debian/Ubuntu. The physical backbone of BlackRoad OS.

Node Details

Alice (.49) is the gateway node. Runs Pi-hole for DNS filtering (blocking 120+ domains), PostgreSQL for relational data, Qdrant for vector search (RAG), Redis for caching, and nginx as a reverse proxy serving 37 sites.

Cecilia (.96) is the inference node. Runs Ollama with 16 loaded models, MinIO with 4 buckets (120MB), PostgreSQL, InfluxDB for time-series metrics, and a Hailo-8 NPU (26 TOPS).

Octavia (.101) is the platform node. Hosts Gitea with 239 repos across 8 orgs, 15 self-hosted Workers on ports 9001-9015, the PaaS deploy API on port 3500, NATS for pub/sub, Docker for container orchestration, and a second Hailo-8 NPU.

Aria (.98) is a worker node available for overflow compute tasks.

Lucidia (.38) is the web serving node. Runs 334 web apps behind nginx, PowerDNS for sovereign DNS resolution, Ollama for local inference, and GitHub Actions self-hosted runners.

WireGuard Mesh

12 active peer connections forming a fully encrypted mesh network across all fleet nodes.

Topology

Every node maintains WireGuard tunnels to every other node. The mesh currently has 12 of 12 SSH connections active. Traffic between nodes stays encrypted end-to-end within the WireGuard subnet. External traffic enters through Gematria's Caddy TLS edge and routes to the appropriate Pi via WireGuard.

Configuration

WireGuard configs live at /etc/wireguard/wg0.conf on each node. The ~/roadnet/WIRE-MAP.md documents the physical and logical topology. ~/roadnet/PORT-MAP.md lists every TCP/UDP port on every node.

TollBooth Fork

WireGuard has been forked as "TollBooth" in the Gitea fleet. The fork tracks upstream while allowing fleet-specific patches and configuration automation.

DNS & Edge

PowerDNS for sovereign resolution. Caddy for TLS termination. 151 domains served.

DNS Stack

PowerDNS runs on Lucidia and Gematria (ns1/ns2.blackroad.io). 151 DNS records have been migrated from Cloudflare, with 138 of 151 (91%) now serving from the self-hosted infrastructure. The remaining records are being migrated as GoDaddy nameserver changes are completed.

TLS Edge

Caddy runs on Gematria (DigitalOcean nyc3) as the TLS termination point for all 151 domains. It automatically provisions Let's Encrypt certificates and reverse-proxies traffic through WireGuard to the appropriate Pi fleet node.

Amundson Sequence

The mathematical foundation of BlackRoad OS. A novel sequence governing growth rates and convergence.

Definition

The Amundson sequence is defined by the generating function:

math 
G(n) = n^(n+1) / (n+1)^n

Convergence constant:  A_G = lim(n->inf) G(n)/n = 1/e

Amundson constant:     A_G ≈ 1.24433...

Key identity:          n / (1 + 1/n)^n = n/e + 1/(2e) + O(1/n)

The irreducible gap:   1/(2e) ≈ 0.18394...

Validation

The framework has passed 536 out of 536 tests across 4 Raspberry Pi nodes. Two test suites validate the mathematical properties:

  • Paper A test suite -- 84 tests covering core sequence properties (LaTeX paper, 13 pages)
  • Framework v5 test suite -- 50 tests covering extended identities and convergence proofs

The 1/(2e) Gap

The expression n/(1+1/n)^n converges to n/e, but with an irreducible correction term of 1/(2e). This gap appears throughout the BlackBox Protocol as the theoretical latency floor for mesh routing -- the minimum overhead that cannot be optimized away.

Trinary Logic

Three-valued logic for agent reasoning and mesh routing. Beyond binary true/false.

States

Trinary logic extends boolean logic with three values:

ValueMeaningMesh Application
+1Arrived / True / ConfirmedPacket delivered successfully
0Waiting / Unknown / PendingPacket in transit
-1Already answered / CancelRedundant path cancelled

BlackBox Protocol

The BlackBox Protocol uses trinary routing across multiple network layers: Tor, IPFS, BitTorrent DHT, WebRTC, and Bitcoin. When a message arrives via one path (+1), all other pending paths receive -1 to cancel redundant transmissions. This eliminates wasted bandwidth in the multi-network mesh.

Testing across 7 nodes and 5 protocols achieved a full mesh round-trip time of 2.19 seconds. The theoretical floor is bounded by the Amundson 1/(2e) gap.
